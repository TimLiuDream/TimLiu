# TimLiu 个人主页设计规范

## 1. 设计目标
- 做一个 GitHub 风格的极简个人主页
- 保持克制、干净、信息清晰
- 让页面有一点点隐藏彩蛋和轻动效，但不喧宾夺主

## 2. 设计关键词
- 极简
- 克制
- 信息密度低
- GitHub 感
- 程序员审美
- 低调彩蛋

## 3. 整体风格
- 参考 GitHub 首页的视觉气质：白底、黑字、灰边、干净排版
- 不使用夸张渐变、大面积炫彩、玻璃拟态、霓虹风
- 视觉重点放在字体层级、留白、边框、hover 细节和轻动效

## 4. 色彩规范

### 浅色模式
- 背景：#ffffff
- 主文字：#0d1117
- 次文字：#57606a
- 边框：#d0d7de
- 卡片底色：#f6f8fa
- 强调色：#0969da

### 暗色模式
- 背景：#0d1117
- 主文字：#c9d1d9
- 次文字：#8b949e
- 边框：#30363d
- 卡片底色：#161b22
- 强调色：#58a6ff

## 5. 字体规范
- 中文：Inter / system-ui / PingFang SC / Microsoft YaHei
- 英文与代码感标题：JetBrains Mono 或 SF Mono
- 标题短而硬朗
- 正文行高略大，保证阅读舒适

### 字号建议
- H1：56–72
- H2：28–36
- 正文：16–18
- 辅助文字：13–14

## 6. 布局规范
- 内容区整体居中
- 最大宽度建议：960px ~ 1120px
- 左右留白充足
- 不做复杂分栏和大图首屏
- 内容块之间层次清晰，但不过度设计化

### 页面结构
- 顶部导航
- Hero 首屏
- About 区域
- Contact 区域
- Footer

## 7. 组件规范

### 卡片
- 轻边框
- 圆角 12px 左右
- 轻阴影或不加阴影
- hover 时边框变亮、背景略微提亮

### 按钮
- 主按钮：实心蓝
- 次按钮：描边灰
- hover 只做轻微位移和颜色变化
- 不做夸张弹跳

### 标签 / Badge
- 小号灰底标签
- 用于技术栈、状态、关键词
- 风格接近 GitHub issue label，但更干净

## 8. 动效规范
- 默认时长：150ms ~ 250ms
- easing：ease-out
- 不连续闪烁
- 不大范围位移
- 不影响阅读

### 推荐动效
1. Hero 标题轻微渐入：淡入 + 上移 8px
2. 按钮 hover：轻微上浮 1px，阴影/边框微变
3. 卡片 hover：边框更亮、背景轻微抬升
4. Logo hover：轻微旋转 2–4 度
5. 小图标点击：一次短暂闪烁
6. 页脚极小签名 hover 才显露
7. About 区域终端感文案：光标轻微闪烁

## 9. 彩蛋建议
- `TimLiu` 字样 hover 时出现细下划线扫描
- 头像点击后切换备用 geek 头像
- 页脚放一行极小字：`Built with curiosity.` / `Still tinkering.`
- 某个角落埋一个小号 `⌘` / `<>` / `#` 图标
- 暗色模式下显示一句隐藏文案

## 10. 交互气质
- 安静
- 克制
- 细看有讲究
- 有工程师气
- 不装、不油
- 像认真写代码的人自己做的网站

## 11. 视觉禁忌
- 不要大面积渐变
- 不要霓虹风
- 不要赛博朋克红蓝乱撞
- 不要过度玻璃拟态
- 不要动效过多
- 不要首页塞太满

## 12. 一句话总结
像 GitHub 一样干净克制，但比 GitHub 多一点私人气质和小心思。
